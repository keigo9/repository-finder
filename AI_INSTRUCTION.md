### 2. 機能実装段階

#### 2-1. 検索ページ実装

**活用方法:**

```
「検索ページを実装して。
- Server Componentsを活用
- searchParamsでクエリパラメータを取得
- GitHub APIでリポジトリ検索
- キャッシュ戦略を適切に設定
- エラーハンドリングも入れて」
```

**確認・修正ポイント:**

- Server ComponentsとClient Componentsの分割が適切か
- `fetch`のキャッシュオプション(`cache`, `revalidate`)が設定されているか
- ローディング状態とエラー状態の処理が実装されているか
- API rate limitへの対応があるか

## 🔍 レビュー・修正の進め方

### Claude Code生成後のチェックリスト

1. **型安全性**
   - `any`型を使っていないか
   - 型定義が十分か

2. **パフォーマンス**
   - 不要な再レンダリングが発生していないか
   - キャッシュ戦略が適切か
   - 画像最適化がされているか

3. **エラーハンドリング**
   - try-catchが適切に配置されているか
   - ユーザーに分かりやすいエラーメッセージか

4. **コード品質**
   - 関数が適切なサイズに分割されているか
   - 命名が分かりやすいか
   - コメントが必要な箇所に記載されているか

5. **アクセシビリティ**
   - セマンティックHTMLを使用しているか
   - キーボード操作に対応しているか
