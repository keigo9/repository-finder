# 生成AI（Claude Code）の活用方法

このプロジェクトでは、開発の全工程において Claude Code を活用しました。

## 開発フロー

### 1. 初期実装（課題の読み込み）

**指示方法:**

```
課題のURLを提示して、そのまま実装を依頼
「https://esa-pages.io/p/sharing/17090/posts/2786/ace7035bd3e5b29d39c1.html
こちらの内容を確認して、手順を立ててから実装して」
```

**AIの動作:**

- 課題内容を解析
- `CLAUDE.md` の設定に従ってタスクを分解し、Todoリストを作成
- 実装を順次実行

### 2. 人間による確認と修正指示

**確認項目:**

- 実際にアプリを動かして動作確認
- UI/UXの違和感チェック
- エラーの発生確認

**修正指示例:**

```
「検索結果が表示されない。エラーログを見て修正して」
「検索フォームのデザインを改善して」
「ローディング中のUIを追加して」
```

### 3. コミット（AIによる自動化）

**指示方法:**

```
「コミットして」
```

**AIの動作:**

- `git status` と `git diff` で変更内容を確認
- `CLAUDE.md` のコミット規則に従ってメッセージを生成
  - `feat:` 新機能
  - `fix:` バグ修正
  - `refactor:` リファクタリング
  - `perf:` パフォーマンス改善
  - など
- コミットを実行

### 4. 段階的な品質改善

大枠の実装が完成し、動作するようになった後、以下の観点で段階的に改善を依頼:

#### リファクタリング

```
「コードを見直して、責務ごとに関数を分割して」
「型定義をもっと厳密にして」
```

#### パフォーマンス改善

```
「Next.js 16のPPRを活用してパフォーマンスを改善して」
「キャッシュ戦略を見直して」
```

#### SEO対策

```
「メタデータを追加してSEO対策して」
「OGP対応して」
```

#### セキュリティ強化

```
「Zodでバリデーションを追加して」
「環境変数のチェックを実装して」
「DoS攻撃対策を追加して」
```

## Claude Codeの設定（CLAUDE.md）

プロジェクトルートに `CLAUDE.md` を配置し、AIの動作をカスタマイズ:

### 主な設定内容

**タスク実行手順:**

1. 該当ファイルの内容を理解
2. タスクを分解してTodoを作成
3. Todoを実行

**コミット規則:**

- Conventional Commits形式
- 日本語でメッセージを記述

**コード作成方針:**

- 可読性重視
- 責務ごとに関数を分解

## AI活用のメリット

1. **初期実装の高速化**: 課題を読ませるだけで基本実装が完了
2. **品質の向上**: 段階的な改善指示で、セキュリティやパフォーマンスも考慮
3. **一貫性のあるコミット**: コミット規則に従った自動コミット
4. **反復作業の効率化**: UI調整やエラー修正を自然言語で指示

## 注意点

- AIの出力は必ず人間が確認・テストする
- 複雑なロジックは段階的に指示を出す
- エラーが出たら、ログを見せて修正を依頼する
